## Механика ценообразования на WB

Из шаблона обновления цен:

- **Цена** (без скидки): от 50 до 20 000 000 ₽ — целое число
- **Скидка**: от 0% до 95% — целое число
- **Цена со скидкой** = Цена × (1 − Скидка / 100)
- **Карантин цен**: если новая цена снижена более чем в несколько раз относительно прежней — товар уходит в карантин. Порог: новая цена × 229 < 50 660 (т.е. цена со скидкой ниже ~221 ₽ — риск карантина при резком снижении)
- **Минимальная цена со скидкой**: 25 ₽

### Предупреждения WB при изменении цен
- «Большинство товаров в выбранной категории стоят дороже 209 ₽» — WB предупреждает при цене ниже медианы категории
- «Вы уменьшили цену в несколько раз, товар попадет в карантин цен»

## Структура артикулов

- **Артикул WB** — числовой, присваивается платформой (например, 266789961)
- **Артикул продавца** — собственный код товара (например, KH3016KH3016, или просто «9», «97»)
- **Баркод** — 13-значный EAN (например, 2006872173031)

## WB API — механика (для интеграций)

### Авторизация
- Заголовок: `Authorization: Bearer ВАШ_ТОКЕН`
- Токен создаётся в личном кабинете WB Партнёры
- Типы токенов: Персональный, Сервисный, Базовый, Тестовый

### Базовые домены
- `https://common-api.wildberries.ru` — общие методы, проверка токена
- `https://content-api.wildberries.ru` — карточки товаров
- `https://discounts-prices-api.wb.ru` — цены и скидки [устарело?]
- `https://marketplace-api.wildberries.ru` — склад и остатки [устарело?]
- `https://statistics-api.wildberries.ru` — статистика продаж [устарело?]
- Проверка подключения: `GET /ping`

### Лимиты запросов (Token Bucket)

| Тип токена | Лимит/мин | Интервал | Burst |
|---|---|---|---|
| Персональный | 300 | 200 мс | 20 |
| Сервисный | 300 | 200 мс | 20 |
| Базовый | 150 | 200 мс | 10 |
| Тестовый | 150 | 200 мс | 10 |

### Обработка ошибок API

| Код | Причина |
|---|---|
| 401 | Токен отсутствует, истёк или повреждён |
| 403 | Недостаточно прав у токена |
| 429 | Превышен лимит запросов |
| 202 | Запрос принят, операция асинхронная (нужно опрашивать статус) |

### При ошибке 429
- Читай заголовок `X-Ratelimit-Retry` — там время ожидания в секундах
- `X-Ratelimit-Remaining` — остаток доступных запросов (в каждом ответе)
- Алгоритм: подожди → повтори → при повторной 429 удвой паузу → макс. 3–5 попыток

### Рекомендации при работе с API
- Запрашивай редко меняющиеся данные (склады, категории) раз в несколько часов
- Используй пакетные методы вместо отдельных запросов на каждый товар
- Следи за `X-Ratelimit-Remaining` и замедляйся при приближении к нулю

## WB API — эндпоинты [устарело?]

_Данные из знаний модели до середины 2025 г. Сверить с актуальной документацией перед использованием._

### Цены и скидки — `discounts-prices-api.wb.ru`

| Метод | Путь | Что делает |
|---|---|---|
| `GET` | `/api/v2/list/goods/filter` | Список товаров с текущими ценами и скидками |
| `POST` | `/api/v2/upload/task` | Загрузить новые цены / скидки (асинхронно, возвращает task ID) |
| `GET` | `/api/v2/history/goods/task` | История изменений цен |

### Остатки и склад — `marketplace-api.wildberries.ru`

| Метод | Путь | Что делает |
|---|---|---|
| `GET` | `/api/v3/warehouses` | Список складов WB |
| `GET` | `/api/v3/stocks/{warehouseId}` | Текущие остатки по складу |
| `PUT` | `/api/v3/stocks/{warehouseId}` | Обновить остатки |
| `DELETE` | `/api/v3/stocks/{warehouseId}` | Обнулить остатки |

### Статистика продаж — `statistics-api.wildberries.ru`

| Метод | Путь | Что делает |
|---|---|---|
| `GET` | `/api/v1/supplier/stocks` | Остатки по складам (из отчётов) |
| `GET` | `/api/v1/supplier/orders` | Заказы за период (параметры: `dateFrom`, `flag`) |
| `GET` | `/api/v1/supplier/sales` | Продажи за период |
| `GET` | `/api/v1/supplier/incomes` | Поставки (приходы на склад) |
| `GET` | `/api/v1/supplier/reportDetailByPeriod` | Детализированный финансовый отчёт с комиссиями WB |

### Карточки товаров — `content-api.wildberries.ru`

| Метод | Путь | Что делает |
|---|---|---|
| `POST` | `/content/v2/get/cards/list` | Список карточек с фото, описаниями, характеристиками |
| `POST` | `/content/v2/cards/update` | Обновить карточку |

### Приоритеты для бизнеса

- **`reportDetailByPeriod`** — юнит-экономика: выручка, комиссия WB, логистика, итоговая выплата по каждой продаже. Позволит понять реальную маржу на серьгах со скидкой 40%
- **`orders` + `sales`** — что реально продаётся vs просто лежит на складе
- **`stocks` через API** — автоматический мониторинг нулевых остатков вместо ручных выгрузок
