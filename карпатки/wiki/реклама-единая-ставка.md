# Единая ставка во внутренней рекламе WB

_Источник: «Единая ставка во внутренней рекламе WB.pdf»_

---

## Главный тезис

> Единая ставка — must have для полок.
> Ручная ставка — must have для поиска.
> Комбинируем.

Единая ставка = одна ставка на все ключи и полки. Управлять позициями по кластерам нельзя. Но её главная ценность — **дешёвые рекомендательные полки**.

---

## Зачем нужна единая ставка

- CPM в полках через единую в **1.5–2 раза дешевле**, чем через ручную
- Больше объём показов в полках — система наваливает больше
- Минимум ручной работы — хлопнул ставку и поехал
- Даже крупные «ярдовики», которые раньше крутили только поиск, перешли на полочный трафик

---

## Где единая ставка НЕ работает

Для поисковой выдачи — единая не подходит:
- Нет управления позицией по каждому кластеру
- Нельзя выверить эффективную позицию для каждого ключа
- Даже позиция 1–5 для разных ключей требует разных ставок — без ручной переплачиваете

---

## Сравнение: единая vs ручная

| Параметр | Единая | Ручная |
|---|---|---|
| Полки (рекомендации) | ★ Лучше, дешевле, больше показов | Хуже, дороже |
| Поисковая выдача | Хуже, нет контроля | ★ Лучше, полный контроль |
| Управление кластерами | Нет | Да |
| Новые карточки | Иногда дешевле бустит | Плюс-минус то же |
| Сложность настройки | Минимальная | Высокая, но даёт результат |
| Оптимизация бюджета | Невозможна | Обязательна |

---

## Алгоритм настройки единой ставки на полки

1. Запустить единую ставку с **минимальной ставки**
2. Потихоньку повышать и минусить все ключи, пока показы в полках растут
3. Показы перестали расти → остановить повышение (достигли потолка)
4. Достигли плановых показателей → начать снижать ставку обратно к минимальной
5. Снижать, пока не начинает падать объём — это оптимальная ставка

**Задача:** найти минимальную ставку при которой полки дают нужный объём. Не переплачивать.

---

## Идеальная связка

- Единая ставка → дешёвые полки с большим объёмом показов
- Ручная ставка → точечный поиск с контролем по каждому кластеру
